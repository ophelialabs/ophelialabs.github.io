An advanced retail storefront UI built with Next.js, designed for a modern microservices architecture.

Overview & Tech Stack

This project is a sophisticated e-commerce storefront designed to be the user-facing entry point for a suite of backend microservices. It is built to be scalable, performant, and easily integrated with operational tools.

  • Framework: Next.js (with App Router)
  • Language: TypeScript
  • Styling: Tailwind CSS
  • Architecture: Standalone frontend, ready for Kubernetes-based microservice integration (e.g., Cart, Catalog, Auth services).
  • Advanced Features:
    • AI Agent Integration: A built-in chat component for conversational commerce and support.
    • Observability Hooks: Guidance for integrating monitoring and analytics agents.

Getting Started

Follow these steps to run the retail UI application locally.

  1. Navigate to the project directory: 
    cd Frontend_UI/retail-ui
  2. Install dependencies:

    This command installs all the necessary packages defined in package.json.

    npm install
  3. Run the development server:

    This starts the Next.js application in development mode.

    npm start
  4. Open the application:

    Visit http://localhost:3000 in your browser to see the running application. You can browse the homepage, click on products, and interact with the AI chat agent.

Integrating Agents

This application is designed to be enhanced with various types of "agents" for AI-driven functionality and operational monitoring.

The UI includes a chat widget that connects to a mock API endpoint (/api/chat). To connect this to a real AI, you would modify this backend route.

File to Edit: app/api/chat/route.ts

Instructions:
  1. Choose an AI Provider: Select a service like OpenAI, Google Gemini, Anthropic, or a self-hosted model.
  2. Install the Provider's SDK: For example, npm install openai.
  3. Update the API Route: Modify the POST function to call the real AI service. Securely store your API key as an environment variable.
Example with OpenAI:

// app/api/chat/route.ts
import { NextRequest, NextResponse } from 'next/server';
import OpenAI from 'openai';

const openai = new OpenAI({
  apiKey: process.env.OPENAI_API_KEY, // Use environment variables
});

export async function POST(req: NextRequest) {
  const { message } = await req.json();

  const completion = await openai.chat.completions.create({
    model: "gpt-3.5-turbo",
    messages: [{ role: "user", content: message }],
  });

  const reply = completion.choices[0].message.content;
  return NextResponse.json({ reply });
}

To monitor frontend performance, user behavior, and errors, you can integrate a Real User Monitoring (RUM) agent.

File to Edit: app/layout.tsx

Instructions:
  1. Choose a Monitoring Service: Options include Vercel Analytics (native), Datadog, New Relic, Sentry, etc.
  2. Follow Integration Docs: Each service provides a small code snippet or component to be added to your application's root layout.
Example with Datadog RUM:

First, install the Datadog package: npm install @datadog/browser-rum

Then, initialize it in a client-side component and include it in your layout.


// app/components/DatadogInit.tsx
'use client';
import { datadogRum } from '@datadog/browser-rum';
import { useLayoutEffect } from 'react';

export const DatadogInit = () => {
  useLayoutEffect(() => {
    datadogRum.init({
      applicationId: 'YOUR_APPLICATION_ID',
      clientToken: 'YOUR_CLIENT_TOKEN',
      site: 'datadoghq.com',
      service: 'retail-ui',
      env: process.env.NODE_ENV,
      version: '1.0.0',
      sessionSampleRate: 100,
      sessionReplaySampleRate: 20,
      trackUserInteractions: true,
      trackResources: true,
      trackLongTasks: true,
      defaultPrivacyLevel: 'mask-user-input',
    });
    return () => {
      datadogRum.stopSession();
    }
  }, []);
  return null;
};

Finally, add <DatadogInit /> to your RootLayout in app/layout.tsx.
